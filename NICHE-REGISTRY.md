# Niche Registry

> Single source of truth for every master prompt generated by this repo. **Update this table on every generation.**

Why this exists: master prompts drift across model versions and meta-prompt versions. Without a registry, you lose track of which prompt produced which video, which model generated which prompt, and when a regeneration is overdue.

**Workflow on every new generation:**

1. Save the prompt to `outputs/generated/{niche}-v{N}.md` with YAML frontmatter (see `outputs/generated/README.md`).
2. Add a row to the table below.
3. Set `Status: draft` initially. Promote to `staged` once you've test-run a video. Promote to `deployed` after first published video performs.
4. On regeneration, append a new row — never edit history.

## Status definitions

- **draft** — generated, not yet test-run. Do not use for production batches.
- **staged** — at least 1 video produced + reviewed internally, not yet published.
- **deployed** — at least 1 video published; performance data available.
- **deprecated** — superseded by a newer version. Keep the row, mark status, link successor in Notes.

## Version bump rules

- **Patch** (`v1.0.1`) — typo fix, single-rule tightening. Same INPUT, same model.
- **Minor** (`v1.1.0`) — `flip mode`, `variant`, `tighten`, `loosen` applied. Same INPUT, same model.
- **Major** (`v2.0.0`) — different INPUT (new reference video, language change), or model upgrade (Opus 4.6 → 4.7).
