# outputs/generated/

This directory holds **every master prompt generated by `META-PROMPT.md`**. One file per niche per version.

## Naming convention

```
{niche}-v{N}.md
```

Examples:
- `mansion-restoration-v1.md`
- `grimdark-history-v2.md`
- `luxury-watch-v1.md`
- `asmr-knife-forging-v1.md`

Use kebab-case for the niche slug. Bump `v{N}` on every regeneration — never overwrite a deployed prompt.

## Required YAML frontmatter

Every generated file MUST start with this block (so the prompt is self-describing and the registry stays in sync):

```yaml
---
niche: grimdark cinematic AI history
youtube_url: https://www.youtube.com/watch?v=X-UgHOce2kk
generated: 2026-04-25
model: claude-opus-4-7[1m]
meta_prompt_version: 2.1.0
input_hash: a3f9c12b
language: English
target_tool: VEO 3.1
duration: longform
script_mode: full-narration
status: deployed
---
```

The `input_hash` is the SHA-256 of the INPUT block (first 8 hex chars). Used to detect when "regenerate" actually has new inputs vs same-inputs-different-output drift. Until v2.2 lands the deterministic lock, treat hash collisions with different outputs as a known-non-determinism event.

## Where to register

Every file added here MUST also be added to `NICHE-REGISTRY.md` at the repo root. Single row per generation. See that file for the schema.
